Connect to EC2 instance via SSH

  1. Open Instance summary by click on Instance ID

2. Click Connect (in right) as shown above.

3. Connect to instance window will open with 4 tabs. Click the SSH client tab.

4. Perform the steps as mentioned in your SSH client tab.

Step 1 — The SSH client in your machine can be terminal, command line or any other tool like putty.

Step 2 — Go to the folder from the terminal where the .pem file was downloaded.

Step 3 — Run command chmod 400 ub-key-pair.pem

Step 4 — Run command ssh -i “ub-key-pair.pem” <address from you ssh client window>

Once the client SDK is generated, you can import and use it in folder hooks, creating a corresponding function that will make a request and return the required data.

For example file hooks/useLocations.ts, you need to take a list of locations: 

import { useAPIGetLocations } from "../../orval_api/location/location";

//  the hook is generated by orval based on our Fast api, with the help of which we take the list of locations

import { useHeadersToken } from "./useHeadersToken";

export const useLocations = () => {

  const { headers } = useHeadersToken();

 // since this request is protected, almost all requests are protected, you need to pass a token in the headers

// for taking the token, there is a separate hook that takes the token from the storage (the token gets into  

// the storage every time the user logs in)

  const { data, isError, isLoading } = useAPIGetLocations({ axios: { headers } });

  return {

    locations: data?.data,

    isLoadingLocations: isLoading,

    isErrorLocations: isError,

  } as const;

};

then we use this function (hook) - useLocations, where we need to display the taken locations in components/Jobs/CreateJob.service.tsx,  the useCreateJob.tsx file is a file in which all logic, i.e. all data manipulation for the CreateJob.tsx component, is transferred!

Alembic allows developers to manage changes to their application's database schema over time, making it easier to evolve and maintain the database schema as the application grows and changes.

Automatic Generation:

Alembic can automatically generate migration scripts based on the changes made to the SQLAlchemy models in the Flask application. This is done using the flask db migrate command, which inspects the current state of the database and generates a migration script to bring it in sync with the changes in the models.

Applying Migrations:

Once a migration script is generated, it can be applied to the database using the flask db upgrade command. This command executes the migration script, applying the specified changes to the database schema. Alembic keeps track of which migrations have been applied, ensuring that each migration is applied only once and in the correct order.

Rollbacks:

Alembic also supports rolling back migrations in case of errors or the need to revert to a previous database schema. This is done using the flask db downgrade command, which reverses the effects of the most recent migration.

Example Usage:

Here's a basic example of using Alembic migrations in a Flask application:

flask db migrate -m "create user table"

flask db upgrade

In this example:

  • the flask db migrate command generates a migration script based on changes to the SQLAlchemy models;

  • the -m flag allows you to provide a message describing the purpose of the migration;

  • the flask db upgrade command applies the generated migration script to the database;

  • src: the directory where you find files for Flask, the connection of the Tailwind and Flowbit libraries, as well as js functions for manipulating actions on Flask templates

  • test_api and test_flask: in these directories there are tests for the Fast API and, accordingly, for Flask
